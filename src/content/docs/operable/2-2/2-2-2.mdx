<Aside title="검사 항목">
자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.
</Aside>

## 목적
자동으로 움직이거나 갱신되는 콘텐츠를 제공하는 경우, 사용자가 직접 정지, 일시 중지 또는 이전/다음으로 이동할 수 있는 기능을 제공해야 한다. 이를 통해 시각 및 인지 장애가 있는 사용자를 포함한 모든 사용자가 충분한 시간 동안 콘텐츠를 읽고 활용할 수 있도록 한다.

##	배경 및 필요성
자동 스크롤이나 자동 갱신되는 콘텐츠는 읽기 속도가 느리거나 시각적 변화를 따라가기 어려운 사용자에게 큰 어려움을 줄 수 있다.

사용자가 콘텐츠 변경을 제어할 수 있는 수단이 없으면, 필요한 정보를 놓치거나 잘못 해석할 위험이 있다. 화면낭독프로그램 사용자는 자동 변경되는 콘텐츠로 인해 현재 읽고 있던 내용이 중단되거나, 초점이 강제로 이동하여 혼란을 겪을 수 있다.

따라서, 자동으로 변경되는 콘텐츠를 제공할 때 반드시 사용자가 이를 정지하거나 일시 중지, 이전/다음 등 이동할 수 있는 수단을 제공해야 한다.

##	기본 규칙
- 기본적으로 자동으로 움직이거나 갱신되는 콘텐츠의 사용을 피해야 한다. 불가피하게 자동 변경되는 콘텐츠를 제공해야 하는 경우, 사용자가 이를 직접 제어할 수 있는 기능을 제공해야 한다.
- 콘텐츠가 자동으로 움직이거나 스크롤 되는 경우, 사용자가 이를 정지하거나 일시 중지할 수 있는 버튼 또는 링크를 제공해야 하며, ‘이전’ 및 ‘다음’ 버튼도 함께 제공하여 원하는 콘텐츠로 직접 이동할 수 있도록 해야 한다.
- 제공된 제어 수단은 키보드와 화면낭독프로그램 등 보조 기술에서도 접근 할 수 있도록 구현해야 한다.
- (권고) 각 제어 버튼에는 기능 설명이 포함된 텍스트나 아이콘을 함께 제공하여, 사용자가 해당 기능의 목적과 사용 방법을 쉽게 이해할 수 있도록 한다.

## 적용 사례
### 1)	시각적으로 보이는 제어수단 제공 (정지, 재생, 이전, 다음 버튼)
자동으로 변경되는 콘텐츠(예: 케러셀, 롤링 공지 등)에 정지, 재생, 이전, 다음 버튼을 제공하여 사용자가 직접 콘텐츠의 변화를 제어할 수 있다.

제어 수단은 화면에서 쉽게 식별할 수 있어야 하며, 모든 사용자가 직관적으로 이해하고 접근할 수 있도록 제공해야 한다. 제어 수단은 키보드 및 보조 기술에서도 인식되고 사용할 수 있도록 구현해야 한다.

또한, 콘텐츠가 정지된 상태에서 ‘이전’ 또는 ‘다음’ 버튼을 눌렀을 때, 정지 상태가 유지되도록 해야 한다.

## 개선 필요 사례
### 1)	제어 수단 미제공
배너가 자동으로 변경되지만, 재생, 정지, 이전, 다음 버튼과 같은 제어수단이 제공되지 않는 사례이다.

사용자가 특정 배너의 내용을 읽는 도중 다음 배너로 자동 전환되어, 사용자는 이전 내용을 충분히 이해하지 못할 수 있다.

특히, 화면낭독프로그램 사용자가 이전 배너를 읽는 도중 초점이 강제로 이동하면, 읽고 있던 내용이 중단되어 현재 위치를 잃거나 탐색 흐름이 방해될 수 있다.

#### 개선 방법
자동 변경되는 배너는 반드시 정지, 재생, 이전, 다음 버튼을 제공해야 한다.

또한, 키보드 입력을 활용한 제어 방식도 제공할 수 있다. 예를 들어, 콘텐츠 내부 요소가 초점을 얻을 때 자동 변경을 일시 중지하고, 화살표(←, →) 키를 사용하여 이전 배너와 다음 배너로 이동하도록 설계할 수 있다.

PC 환경에서는 이러한 키보드 제어 방식이 버튼을 통한 조작과 동일한 효과를 얻을 수 있지만, 모바일 환경까지 고려하면 물리적인 제어 버튼이 필수적이다.

### 2)	키보드 제어 미제공
자동으로 변경되는 ‘오늘의 주요 소식’ 영역에 마우스를 올리면(`hover`) 정지되지만, 키보드 초점(`focus`)을 받을 때는 동일한 정지 기능이 적용되지 않은 사례이다.

이 경우, 키보드 사용자나 화면낭독프로그램 사용자는 콘텐츠가 계속 자동으로 변경되어 현재 내용을 따라가기 어렵고, 초점이 사라지는 불편함을 겪는다.

	![](/images/image158.png)
	<figcaption>이미지. 키보드 초점을 얻고 있는 상태에서 자동 변경이 멈추지 않는 사례</figcaption>

#### 개선 방법
마우스나 키보드로 초점을 받은 상태에서 동일하게 자동 변경이 일시 중지되도록 구현하여, 모든 입력 방식에서 일관된 동작을 보장한다. 또한, 키보드 사용자가 콘텐츠를 탐색하는 동안 정지 상태가 유지되도록 하며, 키보드 초점이 벗어나면 자동 변경이 다시 활성화되도록 구현해야 한다.
