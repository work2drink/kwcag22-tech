입력 오류를 정정할 수 있는 방법을 제공해야 한다.
</Aside>

## 목적
사용자가 입력 오류를 쉽게 인지하고, 어떤 부분이 잘못되었는지 명확한 오류 안내와 정정 방법을 제공하여 오류를 파악하고 수정할 수 있도록 돕기 위한 것이다. 오류 정정 기능을 제공함으로써, 사용자는 입력 과정에서 발생할 수 있는 실수를 빠르게 수정하고 원활하게 서비스를 이용할 수 있도록 지원한다.

##	배경 및 필요성
입력 오류가 발생했을 때, 사용자는 무엇이 잘못되었는지 빠르게 파악하고 이를 수정할 수 있어야 한다. 장애가 있는 사용자는 입력 양식을 작성하는 데 비장애인보다 더 많은 시간이 소요될 수 있고, 오류 안내가 부족할 경우 서비스 이용을 포기할 가능성도 있다.

오류 안내는 화면낭독프로그램과 같은 보조 기술이나 사용자 에이전트(웹 브라우저 등)가 오류 정보를 식별하여 전달할 수 있도록 텍스트 형태로 제공되어야 한다. 또한, 사용자가 오류를 쉽게 인지하고 수정할 수 있도록 텍스트 외에도 이미지, 색상, 기호 등 다양한 시각적 요소를 보조 수단으로 활용할 수 있다. 단, 시각적 정보만으로 오류를 전달해서는 안 되며, 모든 사용자가 접근할 수 있는 방식으로 정보를 제공해야 한다.

더불어, 인지 능력에 제한이 있거나 시각적 정보를 활용하기 어려운 사용자도 오류를 수정할 수 있도록 지원해야 하며, 오류 메시지에는 구체적인 정정 방법과 오류가 발생한 위치 안내가 포함되어야 한다. 예를 들어, '이름 입력란은 공백으로 둘 수 없습니다. 이름을 입력해 주세요.'와 같은 명확한 지침을 제공하고, 이름 입력란에 초점이 자동으로 이동하도록 하여 사용자가 문제를 확인하고 해결할 수 있도록 도울 수 있다.

##	기본 규칙
- 입력 오류 발생 시, 사용자가 즉시 오류를 인지할 수 있도록 명확한 오류 메시지를 제공해야 한다. 오류 안내는 전체 입력 양식에 걸쳐 일관되게 적용되어야 한다.<br/>예) 생년월일 입력란에서 '생년월일은 8자리로 입력해 주세요.'와 같이 구체적인 안내를 제공한다.
- 오류 메시지는 사용자가 오류를 어떻게 정정할 수 있는지 명확한 지침을 포함해야 한다.<br/>예) '오류가 있습니다.'라는 메시지 대신 '비밀번호는 최소 8자 이상이며, 숫자와 특수문자를 포함해야 합니다.'와 같이 구체적인 요구사항을 명시한다.
- 오류 정보는 색상만으로 제공하지 않으며, 시각적 요소(색상, 기호, 이미지 등)는 오류 정보를 보완하는 수단으로 활용할 수 있다.<br/>예) 필수 입력 항목을 색상(빨간색 테두리)만으로 표시하지 않아야 한다.
- 오류가 발생한 입력 요소에 초점을 이동시켜, 사용자가 오류 위치를 쉽게 확인하고 수정할 수 있도록 해야 한다.

## 적용 사례
### 1)	오류 안내 제공
회원가입 시 입력한 아이디가 조건에 맞지 않는 경우 '아이디를 5자 이상 입력하세요'와 같이 오류의 원인과 위치를 안내하여 사용자가 오류를 식별하고 수정할 수 있도록 구체적으로 정보를 제공한다.

![이미지. 오류의 구체적인 내용을 텍스트 정보로 제공한 사례 (출처: 서울시)](/images/image232.png)

다음 사례는 오류가 발생한 입력란에 시각적인 색상 정보와 함께 오류 안내 텍스트를 제공하였다.

![이미지. 오류를 쉽게 인식하고 수정할 수 있도록 붉은 테두리와 정정 방법을 안내한 사례 (출처: 토스)](/images/image233.png)

### 2)	오류 위치로 초점 이동
입력 오류가 발생하면 일부 사용자는 오류를 찾고 수정하는 데 어려움을 겪을 수 있다. 특히, 화면낭독프로그램 사용자나 인지적 제한이 있는 경우, 오류 위치를 기억하거나 다시 찾는 데 시간이 많이 소요되어 불편함이 커질 수 있다.

오류가 발생한 위치로 초점을 이동시키면 사용자가 오류를 즉시 인지하고 수정할 수 있으며, 불필요한 탐색 과정을 줄여 효율적으로 문제를 해결할 수 있다.

![이미지. 오류 확인 후 해당 입력란으로 초점 이동 사례 (출처: 다음 고객센터)](/images/image234.png)

### 3)	(권고) 입력 양식에 대한 설명 및 예시 제공
입력란에 특정 형식의 값(예: 날짜, 이메일, 전화번호 등)을 요구하는 경우, 사용자가 입력 형식을 쉽게 이해할 수 있도록 구체적인 설명이나 예시를 제공한다.

입력 형식에 대한 명확한 안내는 사용자에게 올바른 값을 입력하도록 유도하여 오류를 줄이고, 불필요한 재입력을 방지한다. 

예를 들어, 생년월일을 '19950301 형식으로 입력하세요' 또는 '`example@example.com`과 같은 이메일 주소를 입력하세요'와 같이 안내하여 사용자가 요구사항을 명확히 이해할 수 있도록 한다.

![이미지. 회원 가입 시 특정 입력 형식에 대해 사전 안내한 사례 (출처: 서울 시청)](/images/image235.png)

## 개선 필요 사례

### 1)	오류 안내 미제공
회원가입에 필요한 모든 정보를 입력하고 회원가입 버튼을 눌렀으나 회원가입이 진행되지 않고 입력 오류에 대한 안내가 제공되지 않는 사례이다.

오류 안내가 없으면 사용자는 어떤 부분에서 문제가 발생했는지 알기 어려워, 이를 수정하지 못해 회원가입 절차를 완료하지 못할 가능성이 높아진다.

![이미지. 입력값에 오류가 있지만 아무런 안내도 제공되지 않는 예시](/images/image236.png)

#### 개선 방법
입력 오류가 발생했을 때 사용자에게 명확하고 접근할 수 있는 방식으로 오류를 안내해야 한다. 우선, 오류가 발생한 입력란에 근처에 텍스트로 오류 메시지를 제공하여 사용자가 쉽게 확인할 수 있도록 해야 한다.

### 2)	오류 발생 영역으로 초점 미이동
입력 오류 발생 시, 오류가 발생한 입력란으로 초점이 이동되지 않아 화면낭독프로그램 사용자나 운동 장애 사용자가 오류 위치를 찾기 어려운 사례이다.

이에 따라 사용자는 오류 위치를 찾기 위해 탐색을 반복해야 하고, 운동장애 사용자는 오류가 발생한 입력란으로 이동에 어려움을 겪을 수 있다. 또한, 인지 능력에 제한이 있는 사용자는 오류 메시지와 위치를 연결하기 어려울 수 있고, 저시력 사용자는 오류가 발생한 입력란을 찾기 위해 더 오랜 시간을 들여야 할 수 있다.

![이미지. 입력 오류 발생 시 회원가입 버튼에 초점이 유지되는 사례](/images/image237.png)

### 3)	구체적인 오류 설명 미제공
비밀번호 입력 오류 발생 시, '잘못 입력되었습니다'와 같이 구체적이지 않은 오류 메시지만 제공되는 사례이다.

이 경우, 사용자는 문제의 원인을 파악하기 어려워 같은 오류를 반복할 수 있다. 예를 들어, 입력 형식이 잘못된 경우에도 이를 명확히 안내하지 않으면 사용자는 같은 오류를 반복할 수 있다. 특히, 인지 능력에 제한이 있는 사용자에게는 오류를 수정하는 과정이 더욱 부담될 수 있다.

![이미지. 구체적인 오류 설명이 제공되지 않은 사례](/images/image238.png)

### 개선 방법
입력 오류 시 제공되는 오류 메시지는 문제의 원인을 명확히 설명해야 한다. 예를 들어, '비밀번호는 최소 8자 이상이어야 하며, 숫자와 특수문자를 포함해야 합니다.'와 같이 구체적인 요구 조건을 명시해야 한다.

![이미지. 구체적인 오류 설명을 제공하도록 개선한 예시](/images/image239.png)

### 4)	오류 발생 시 입력했던 값 초기화
회원 가입 입력 내용을 제출했을 때 오류가 발생하여, 입력한 모든 값이 초기화되는 사례이다. 이 경우, 사용자는 이전에 입력했던 데이터를 다시 입력해야 하여 불필요한 시간 소모와 작업 반복이 발생한다.

특히, 키보드만 사용하는 사용자에게는 입력란을 다시 탐색하고 값을 입력하는 과정에서 추가적인 작업이 요구되어 작업 시간이 증가할 수 있다. 

이러한 경우는 입력 과정에서 동일한 오류를 반복하게 만들며, 결과적으로 사용자가 회원가입을 완료하기 어렵게 만든다.

![이미지. 오류 발생 시 입력값 전체가 초기화되는 사례](/images/image240.png)

#### 개선 방법
입력 오류 발생 시, 오류가 있는 입력란만 강조하고 다른 입력란의 값은 유지되도록 하여 사용자가 불필요하게 데이터를 반복 입력하지 않도록 해야 한다.
