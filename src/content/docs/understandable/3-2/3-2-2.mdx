<Aside title="검사 항목">
[도움 정보](../../glossary/help)가 제공되는 경우, 각 페이지에서 [동일한 상대적인 순서](../../glossary/same-relative-order)로 접근할 수 있어야 한다.
</Aside>

## 목적
사용자가 웹 사이트를 이용하다가 도움이 필요할 때, 전화번호, 챗봇, FAQ, 고객센터 등의 도움 정보를 어디서든 쉽게 찾고 접근할 수 있도록 돕기 위한 것이다.

##	배경 및 필요성
도움 정보가 여러 페이지에서 일관된 위치에 제공되지 않으면, 사용자가 필요한 지원이나 안내를 찾는 데 어려움을 겪을 수 있다. 특히, 다음과 같은 사용자에게 불편을 초래할 수 있다.
- 화면낭독프로그램 사용자: 페이지마다 도움 정보 위치가 다르면, 매번 페이지 전체를 탐색해야 하므로 시간과 노력이 크게 든다.
- 화면 확대를 사용하는 저시력자: 도움 정보를 찾기 위해 화면을 계속 스크롤 하거나 이동하며 탐색해야 하므로 피로도가 증가한다.
- 인지 능력에 제한이 있는 사용자: 도움 정보 위치가 변할 경우, 새로운 위치를 기억해야 하며, 원래 하려던 작업을 잊을 수도 있다.
- 학습 능력에 제한이 있는 사용자: 웹 사이트 구조를 학습하는 데 시간이 더 걸리며, 일관성이 없으면 혼란을 느낄 수 있다.

이처럼 도움 정보가 페이지마다 다른 위치에 있으면 사용자는 불안감을 느끼거나 사이트 이용을 포기할 가능성이 커진다. 반면, 페이지 구성이 달라져도 도움 정보가 항상 동일한 영역 또는 동일한 마크업 순서에 배치되면, 사용자 만족도와 접근성이 모두 향상된다.

##	기본 규칙
- 페이지마다 반복되는 도움 정보는 일관된 위치에 배치되어야 한다.
- 도움 정보의 내용(FAQ, 고객센터 연락처, 챗봇 호출 버튼 등)과 표시 형식을 모든 페이지에서 일관되게 유지한다.

<Card title="참고. 일관된 위치에 대해" icon="open-book">
'일관된 위치'란 단순히 시각적 위치뿐만 아니라, HTML 마크업 구조에서도 동일한 순서로 제공되는 것을 의미한다. 예를 들어, 도움 정보가 항상 사이트 상단 영역(header) 마지막 항목으로 제공되거나, 사이트 하단 영역(footer)의 첫 번째 항목으로 일관되게 배치되는 방식이 이에 해당한다.
</Card>

## 적용 사례
### 1)	연락처를 일관된 위치에 제공
도움 정보를 일관된 위치에 배치하여, 사용자가 필요할 때 예측 가능한 위치에서 쉽게 찾을 수 있도록 한다.
#### a)	모든 페이지 하단에 연락처 정보를 일관된 위치와 형태로 제공
홈페이지의 모든 페이지 하단(Footer)에 ‘고객센터 문의’ 버튼 및 연락처 정보를 동일한 마크업 순서와 시각적 위치에 배치하였다. 이를 통해 사용자는 어느 페이지에서든 동일한 방식으로 도움 정보를 찾을 수 있어, 페이지 탐색 부담이 크게 줄어든다.

#### b)	챗봇 실행 버튼을 모든 페이지에 동일한 위치에 제공
쇼핑몰 사이트의 하단 우측에 고정된 위치에 챗봇 호출 버튼을 두고, 마크업에서도 하단(footer) 영역의 첫 번째 요소로 제공하였다. 시각적으로도 항상 화면 아래 우측에 나타나고, 화면낭독프로그램 사용자는 하단(footer) 영역으로 이동했을 때 첫 항목에서 챗봇 기능을 바로 찾을 수 있다.
쇼핑몰 사이트에서는 하단 우측에 챗봇 호출 버튼을 고정된 위치에 배치하고, 마크업에서도 footer 요소 내 첫 번째 요소로 제공하였다. 이에 따라, 시각적으로는 항상 화면 하단 우측에 챗봇 버튼이 나타나며, 화면낭독프로그램 사용자도 하단(footer) 영역으로 이동했을 때 첫 번째 항목에서 챗봇 기능을 쉽게 찾을 수 있다.

## 개선 필요 사례

### 1)	페이지마다 다른 실시간 채팅 버튼의 위치
오픈마켓에서는 실시간 채팅 버튼이 페이지 진입 때마다 광고 배너의 위치나 배치에 따라 시각적 위치가 달라지는 문제가 발생하고 있다. 마크업 순서는 동일하더라도 화면상 노출 위치가 변경되면, 사용자는 기억하고 있던 위치에서 채팅 버튼을 찾을 수 없어 재탐색해야 하는 불편을 겪게 된다.

이로 인해 페이지마다 버튼의 위치를 다시 확인해야 하며, 특히 시각적 단서를 활용하는 사용자는 더욱 혼란을 느낄 수 있다.

#### 개선 방법
페이지가 변경되어도 실시간 채팅 버튼을 고정된 영역에 배치하고, 마크업 순서뿐 아니라 시각적으로도 동일한 위치를 유지하도록 제공해야 한다. 

이때, 광고 등 다른 요소가 도움 정보의 위치에 영향을 주지 않도록 고려하여 디자인해야 하며, 사용자가 예측할 수 있는 위치에서 항상 채팅 버튼을 찾을 수 있도록 해야 한다.

### 2)	인적 연락처 정보의 위치가 페이지마다 다른 경우
메인 페이지에서는 고객센터 정보가 오른쪽 사이드 영역에, 서브 페이지에서는 하단 영역에 배치되는 등 일관성이 없이 제공되는 사례이다. 이에 따라 사이트를 이용하는 과정에서 사용자가 혼란을 느끼고, 추가적인 탐색 시간이 발생할 수 있다.

#### 개선 방법
모든 페이지에서 반복되는 인적 연락처 정보를 동일한 시각적 위치 및 마크업 구조로 제공한다.
