<Aside title="검사 항목">
페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.
</Aside>

## 목적
페이지, 프레임, 콘텐츠 블록에 적절한 제목을 제공하여 사용자가 웹 페이지의 내용과 구성, 각 섹션의 역할을 쉽게 파악할 수 있도록 한다. 특히, 화면낭독프로그램 같은 보조 기술을 사용하는 경우, 제목 정보가 빠른 탐색과 원하는 영역으로의 이동을 지원하여 전체 문서의 맥락을 이해하는 데 도움을 준다.

##	배경 및 필요성
화면낭독프로그램 사용자는 HTML 문서의 선형 구조를 따라 페이지를 순차적으로 탐색해야 하므로, 제목 정보가 없으면 모든 내용을 일일이 확인해야 하는 부담이 발생한다.

문서 제목으로 `head` 요소 내 `title` 요소, 제목을 나타내는 h1~h6 요소 그리고 프레임의 `title` 속성을 활용하면 보조 기술이 문서의 구조를 쉽게 파악하고, 사용자가 원하는 정보를 빠르게 찾고 이동할 수 있다.

페이지 제목은 콘텐츠의 주제, 목적, 내용 및 구성 간의 관계를 명확하게 전달하며, 사용자가 전체적인 문맥을 쉽게 이해할 수 있도록 한다. 예를 들어, 각 웹 페이지가 고유한 문서 제목을 가지고 있으면, 사용자가 브라우저 탭을 여러 개 열어둔 경우에도 쉽게 페이지를 구분할 수 있다.

##	기본 규칙
이를 위해 다음의 사항들이 제공되어야 한다.
- 모든 페이지는 `title` 요소를 사용하여 페이지의 주제와 내용을 명확하게 나타내야 하며, 사이트 내에서 고유해야 한다.
- `frame`이나 `iframe` 요소에는 `title` 속성을 사용하여 프레임의 용도와 내용을 간결하게 설명해야 한다. 프레임에 포함된 콘텐츠를 식별할 수 있도록 제공하되, ‘~ 프레임’과 같이 불필요한 단어는 사용하지 않는다.
- 개발 용도 등으로 프레임에 사용자에게 전달할 콘텐츠가 없는 경우, `title="내용없음"` 또는 `title="빈프레임"`으로 작성하여 사용자가 탐색하지 않도록 한다.
- 콘텐츠 블록에는 `h1`~`h6` 요소를 사용하여 계층 구조를 고려한 제목을 제공한다.
- 디자인상의 이유 등으로 콘텐츠 블록의 제목을 시각적으로 숨겨야 할 때에도, 보조 기술에서 인식할 수 있도록 적절한 기법을 사용한다.
- (권고) 콘텐츠 블록의 제목은 시각적으로도 명확하게 구분되어야 하며, 화면낭독프로그램 사용자가 제목만 듣고 각 콘텐츠 블록의 역할과 내용을 이해할 수 있도록 구성한다.

## 적용 사례
### 1)	페이지 제목 제공
각 페이지는 title 요소를 사용하여 페이지의 주제와 내용을 명확하게 나타내고, 사이트 내에서 고유해야 한다. 웹 페이지의 제목은 현재 페이지 정보와 사이트 이름을 간결하게 표기하며, 페이지 경로 안내(예: 홈>민원 서비스>민원 신청, 안내) 형식으로 작성하지 않는다. 이는 사용자가 먼저 확인해야 할 중요한 페이지 정보가 뒤에 제공되는 것을 방지하기 위함이다.

<figure>
	![](/images/image168.png)
	<figcaption>이미지. 페이지 제목을 페이지 주요 정보로 제공한 사례 (출처: 정부24)</figcaption>
</figure>

```html
<title>민원 신청·안내 | 정부24</title>
```

### 2)	콘텐츠 블록 제목 제공
각 콘텐츠 블록에 적절한 제목을 제공하여, 보조 기술 사용자가 콘텐츠 구조를 쉽게 이해하고 효율적으로 탐색할 수 있도록 해야 한다. 잘 구성된 제목 구조는 페이지의 목차 역할을 하며, 사용자가 주요 콘텐츠를 빠르게 찾는 데 도움을 준다.

#### a)	시각적으로 보이는 콘텐츠 블록 제목 제공
시각적으로 콘텐츠와 구분되는 제목은 가독성을 높이고, 콘텐츠 탐색 효율성이 향상된다. 또한, 제목 요소(h1~h6)를 사용하여 콘텐츠 블록의 계층 구조를 명확히 표현해야 한다.

<figure>
	![](/images/image169.png)
	<figcaption>이미지. 시각적으로 구별되는 콘텐츠 블록 제목 제공 사례 (출처: 국회도서관)</figcaption>
</figure>

#### b)	시각적으로 숨겨진 콘텐츠 블록 제목 제공
제목이 화면에 노출되지 않더라도, 보조 기술에서 인식할 수 있는 CSS 기법을 사용하여 제공할 수 있다. display:none, visibility:hidden, font-size:0 등의 속성은 화면에서 숨길 뿐만 아니라 보조 기술에서도 인식되지 않으므로 사용하지 않는다. 가능하면 제목은 시각적으로도 보이도록 제공하는 것이 모든 사용자에게 유리하다.

<figure>
	![](/images/image170.png)
	<figcaption>이미지. 시각적으로 생략된 콘텐츠 블록의 제목 제공 사례 (출처: 다음)</figcaption>
</figure>

CSS는 아래와 같이 설정해 볼 수 있다.

```css
.visually-hidden {
	clip-path: inset(100%);
	clip: rect(1px, 1px, 1px, 1px);
	height: 1px;
	overflow: hidden;
	position: absolute;
	white-space: nowrap;
	width: 1px;
}
```
### 3)	프레임 제목 제공
`frame` 이나 `iframe` 요소에는 `title` 속성을 사용하여 해당 프레임의 용도와 내용을 간결하고 명확하게 설명해야 한다. 이는 보조 기술 사용자가 해당 프레임 영역을 탐색할지를 판단하는 데 도움을 준다.

<figure>
	![](/images/image171.png)
	<figcaption>이미지. 아이프레임으로 제공된 콘텐츠 영역 예시</figcaption>
</figure>

```html
<iframe src="..." title="정책브리핑 뉴스 위젯"></iframe>
```

## 개선 필요 사례

### 1)	페이지 제목 미제공
페이지 제목이 없으면 사용자는 페이지의 주제와 목적을 파악하기 어려워지고, 여러 개의 탭을 열어둔 경우에도 각 탭의 내용을 구분하기 어려워진다.

제목 요소가 제공되지 않거나 내용이 없는 경우 브라우저는 URL을 대신 표기하지만, URL만으로는 페이지 정보를 쉽게 파악할 수 없다.

<figure>
	![](/images/image172.png)
	<figcaption>이미지. 페이지 제목이 제공되지 않아 URL이 대신 표시된 사례</figcaption>
</figure>

```html
<html lang="ko">
<head>
	<meta charset="utf-8">
	<link rel="stylesheet" href="/css/style.css">
	<script>...</script>
	...
</head>
```

#### 개선 방법
각 페이지에 고유하고 구체적인 제목을 title 요소로 제공한다.

### 2)	모두 동일한 페이지 제목
모든 페이지 제목이 동일하면, 사용자는 페이지 이동 여부를 파악하기 어렵고, 여러 개의 탭이 열린 상황에서 페이지를 구분하기 어려워진다.

<figure>
	![](/images/image173.png)
	<figcaption>이미지. 홈, 신상품, 베스트 페이지에 동일한 제목이 제공된 사례</figcaption>
</figure>

```html
<!-- 홈 페이지 -->
<title>스타일마켓</title>
<!-- 신상품 페이지 -->
<title>스타일마켓</title>
<!-- 베스트 페이지 -->
<title>스타일마켓</title>
```

#### 개선 방법
페이지마다 주제를 명확히 나타내는 고유한 제목을 제공하며, 페이지 주요 정보가 앞에 위치하도록 간결하게 작성해야 한다.

<figure>
	![](/images/image174.png)
	<figcaption>이미지. 홈, 신상품, 베스트 페이지에 맞는 제목이 제공된 사례</figcaption>
</figure>

```html
<!-- 홈 페이지 -->
<title>스타일마켓</title>
<!-- 신상품 페이지 -->
<title>신상품 | 스타일마켓</title>
<!-- 베스트 페이지 -->
<title>베스트 | 스타일마켓</title>
```

### 3)	제목 요소로 구현되지 않은 콘텐츠 블록 제목
시각적으로 제목처럼 보이지만, h1~h6 요소가 아닌 strong 요소로 구현된 사례이다. 이 경우, 보조 기술은 이를 일반 텍스트로 인식하여 제목으로 처리하지 않으므로, 화면낭독프로그램 사용자는 제목 단위 탐색이 불가능하고 콘텐츠 구조를 파악하기 어려워진다.

<figure>
	![](/images/image175.png)
	<figcaption>이미지. 콘텐츠 블록 제목이 h1~h6 요소로 제공되지 않은 사례</figcaption>
</figure>

```html {2-4}
<div class="_title_area">
<strong class="_title">
    주목할 최신곡 <svg ... class="_icon_more"> ... </svg>
  </strong>
  <a href="..." class="_button_more">더보기</a>
</div>
...
```

#### 개선 방법
화면낭독프로그램 사용자가 제목을 올바르게 인식하고 콘텐츠 구조를 탐색할 수 있도록, h1~h6 요소를 사용하여 제목을 구현해야 한다.

```html {2-4}
<div class="_title_area">
	<h3 class="_title">
    주목할 최신곡 <svg ... class="_icon_more"> ... </svg>
  </h3>
  <a href="..." class="_button_more">더보기</a>
</div>
...
```

### 4)	콘텐츠 블록에 일부 제목 미제공
콘텐츠 블록에 제목이 제공되었지만, 일부 블록에 제목이 빠진 사례이다. 각 콘텐츠 블록에 h3 요소로 제목이 제공되는 반면, 일부 콘텐츠 블록에는 제목이 빠졌다. 

이 경우, 화면낭독프로그램 사용자는 제목이 없는 콘텐츠 블록을 이전 블록과 연결된 것으로 오해할 수 있다.

<figure>
	![](/images/image176.png)
	<figcaption>이미지. 각 콘텐츠 블록(붉은색 영역)에 제목(파란색 테두리)이 일부 제공되지 않은 사례</figcaption>
</figure>

```html {3, 8, 21}
<div id="D2008001072" class="area_lv1">
	<div class="mdPick">
		<h3 class="mnTitle">MD’s PICK ...</h3>
	</div>
</div>
<div id="D2008001235" class="area_lv1">
	<div class="mnCpn">
		<h3 class="mnTitle"><b>바로 사용 가능</b>한 쿠폰이에요! ... </h3>
	</div>
</div>
<!-- 제목 미제공 된 콘텐츠 블록 -->
<div id="D2008001071" class="area_lv1">
	<div class="prmRcm">
		<div class="planLst n2 slide ready">
			...
		</div>
	</div>
</div>
<div id="D2008001074" class="area_lv1">
	<div class="mnBx funIdea">
		<h3 class="mnTitle">FUN & IDEA </h3>
		...
	</div>
</div>
<!-- 제목 미제공 된 콘텐츠 블록 -->
<div id="D2008001075" class="area_lv1">
	<div class="mnBX DFinder">
		...
	</div>
</div>
```

#### 개선 방법
각 콘텐츠 블록에 독립적이고 구체적인 제목을 제공하여, 사용자가 해당 블록의 목적과 내용을 명확히 이해할 수 있도록 해야 한다.

여러 이유로 제목이 시각적으로 노출되지 않더라도, 보조 기술에서 접근할 수 있도록 적절한 방법(예: 화면낭독프로그램 전용 클래스)을 사용한다.

### 5)	프레임 제목 미제공
페이지 내 삽입된 `iframe` 지도 콘텐츠에 제목이 제공되지 않은 사례이다.

`iframe` 요소에 `title` 속성이 누락되면, 화면낭독프로그램 사용자는 해당 프레임의 용도를 파악할 수 없으며, 탐색 및 이해에 불필요한 시간과 노력이 소요된다.

<figure>
	![](/images/image177.png)
	<figcaption>이미지. 지도 콘텐츠를 제공하는 프레임에 제목이 제공되지 않은 사례</figcaption>
</figure>

```html {11}
<div>
	<h2 class="title">...</h2>
	...
	<div class="swiper wrap-roll">
		<ul>
			...
		</ul>
	</div>
</div>
<div class="map">
	<iframe src="..." width="600" height="450" loading="lazy"></iframe>
</div>
```

#### 개선 방법
`iframe` 요소에 프레임의 용도와 목적을 간결하고 명확하게 설명하는 제목을 `title` 속성으로 제공해야 한다.

예를 들어, 지도에 특정 위치를 나타내는 것이 목적이라면, `iframe` 요소에 `title="사무실 위치 안내"`와 같이 설정할 수 있다. 이렇게 설정하면 화면낭독프로그램 사용자가 프레임의 목적을 이해하고 탐색 여부를 판단할 수 있다.
