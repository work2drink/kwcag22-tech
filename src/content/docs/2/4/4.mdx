<Aside title="검사 항목">
전자출판문서 형식의 웹 페이지는 각 페이지로 이동할 수 있는 기능이 있어야 하고, 서식이나 플랫폼에 상관없이 참조 위치 정보를 일관되게 제공ㆍ유지해야 한다.
</Aside>

## 목적
이 검사 항목은 디지털 전용 출판물(전자책)이나 인쇄물을 전자출판문서 형식의 웹페이지로 제공한 경우에만 적용된다. 웹 페이지로 제공된 전자출판문서에 원본과 동일한 각 페이지로 이동할 수 있는 고정된 참조 위치를 제공하는 것이 목적이다. 예를 들어, 종이책과 디지털 출판물을 동시에 참고하거나 특정 페이지의 문단을 찾아야 할 때, 사용하는 디바이스나 글자 크기·줄 간격 설정이 달라도 동일한 위치를 식별할 수 있도록 돕는다.

##	배경 및 필요성
고정된 참조 위치가 필요한 사례로, 대학교에서 인쇄본과 전자책을 함께 사용하는 환경이 있다. 예를 들어, 인쇄본 기준 26페이지의 첫 번째 문단을 찾아야 할 경우, 전자책을 웹에서 볼 때 해상도나 글자 크기 변경으로 인해 페이지 번호가 달라지면 해당 위치를 찾기 어려울 수 있다. 이때, 고정된 참조 위치 정보를 제공하면, 사용자는 동일한 페이지를 쉽게 찾을 수 있다.

이처럼, 고정된 참조 위치 정보가 제공되면, 서로 다른 해상도의 기기(노트북, 패드, 모바일 기기)나 보조 기술을 사용하는 사용자들도 동일한 페이지를 탐색할 수 있다.

고정된 참조 위치 제공이 필요한 콘텐츠 유형은 다음과 같다.
- 웹 페이지로 변환된 전자출판문서를 제공하는 경우
- 인쇄본이 존재하며, 그대로 웹에 보여주고 페이지 정보가 필요한 경우
- 웹 페이지로 변환된 전자출판문서를 다양한 디바이스에서 보거나, 폰트 크기, 글꼴 등 설정 변경으로 인해 페이지가 달라질 수 있는 경우

고정된 참조 위치 정보를 제공하면 다음 사용자에게 도움이 된다. 저시력 사용자 또는 고령자는 화면 확대나 글자 크기를 변경하여 페이지 번호가 달라지더라도, 고정된 위치 정보를 통해 원하는 위치를 찾을 수 있다. 학습 장애 사용자는 인쇄물과 웹 페이지의 페이지 구성이 다르더라도, 고정된 참조 위치 정보를 통해 필요한 위치로 이동할 수 있고, 마찬가지로 주의력결핍 장애 사용자도 페이지 이동 기능이 제공되면 집중력을 유지하면서 맥락을 따라가기 쉬워진다.

해당 검사 항목의 예외 대상은 다음과 같다.
- 별도 애플리케이션 설치 후 실행하여 볼 수 있는 전자출판문서 형태 (예: URL 없이 별도 뷰어 프로그램 사용이 필요한 콘텐츠)
- 웹 페이지로 직접 변환하지 않은 파일 그대로 다운로드하여 제공되는 콘텐츠(예: PDF 파일, Word 파일 등)
- 단일 스크롤 형태 콘텐츠 (예: 한 페이지에서 모든 내용을 제공하는 웹 소설 등, 페이지 구분이 필요 없는 경우)

##	기본 규칙
- 인쇄물이나 전자출판문서를 웹에서 제공하는 경우, 페이지마다 고유한 참조 위칫값을 제공해야 한다. 예를 들어, 대응이 필요한 물리적 페이지 번호나 영역에 id="page26"과 같은 구분자를 부여하여 변하지 않는 위치 정보를 제공한다. 
- 고정된 참조 위치 정보는 사용자가 쉽게 이동할 수 있는 이동 수단(예: 페이지 번호 링크, 페이지 번호 입력창, 드롭다운 목록, '다음/이전' 버튼 등)을 제공한다.
- 보조 기술을 통해서도 접근할 수 있도록 보장된 방법으로 구현해야 한다.

<Card title="참고. 다운로드 받은 PDF 파일" icon="open-book">
PDF 파일을 다운로드하여 브라우저에서 열 경우, 이는 전자출판문서 형식의 웹페이지로 제공된 것이 아니므로 검사 대상에서 제외된다. 즉, PDF 파일 자체를 제공할 때는 고정된 참조 위치 정보 제공이 요구되지 않는다.

## 개선 필요 사례

### 1)	고정된 참조 위치가 제공되지 않은 ePub 뷰어
논문 ePub 파일을 웹페이지 기반 뷰어로 제공한 사례이다. 모니터 해상도, 브라우저 창 크기, 본문 확대/축소, 뷰어 내 설정에 따라 페이지 번호가 계속 변동되어, 결과적으로 ‘논문집 3페이지’와 같은 인쇄본 기준의 위치를 정확히 찾아가기 어렵다.

이동 수단 중 하나로 목차(<abbr title="table of contents">TOC</abbr>)도 제공하였으나, 이는 각 페이지에 대한 고정된 참조 위치를 대체할 수 없으므로 충분한 이동 수단이 될 수 없다.

	<figcaption>이미지. ePub을 웹페이지로 제공하는 뷰어에서 글자 크기 및 줄 간격을 변경할 수 있는 설정 제공 사례</figcaption>

#### 개선 방안
인쇄본과 동일한 고정된 참조 위치 정보를 웹페이지 각 구간에 지정하고, 이를 바탕으로 사용자가 원하는 페이지로 이동할 수 있는 수단(예: 페이지 번호 링크, 이동 버튼 등)을 함께 제공해야 한다. 예로 ‘페이지 번호 입력’ 창을 제공하거나, 목차와 별도로 페이지 단위 바로가기를 구축한다. 사용자가 폰트 크기를 바꿔도 해당 위치가 동일하게 식별되는지 테스트 및 설계해야 한다.

### 2)	고정된 참조 위치 및 이동 수단이 미제공된 보고서
PDF로 발행된 보고서를 웹페이지로 제공한 사례이다. PDF 본문을 이미지로 제공하며, 모든 내용을 하나의 웹페이지에서 스크롤 방식으로 탐색해야 한다. 각 페이지의 참조 위치(페이지 번호)가 시각적으로 표시되어 있지만, 대체 텍스트가 제공되지 않아 보조 기술 사용자(예: 화면 낭독기 사용자)는 이를 확인할 수 없다. 

또한, 고정된 참조 위치 및 해당 페이지로 이동할 수 있는 기능이 없어, 사용자가 일일이 스크롤하여 원하는 내용을 찾아야 한다.

	<figcaption>이미지. PDF 보고서의 각 페이지를 이미지로 변환해 웹페이지로 제공한 사례</figcaption>

#### 개선 방안
페이지마다 고정된 참조 위치 정보를 제공하고, 이를 통해 사용자가 손쉽게 원하는 페이지로 이동할 수 있는 이동 수단을 추가해야 한다. 또한, 이동 수단은 보조 기술에서도 접근할 수 있도록 구현해야 한다.

##	검사 대상이 아닌 사례
### 1)	PDF 파일이 제공되는 뷰어
출판된 도서의 미리보기를 PDF 파일 형태 그대로 제공하는 경우이다.

PDF 파일은 고정된 레이아웃을 가지므로 화면을 확대하거나 축소하더라도 페이지 정보가 변경되지 않고 그대로 유지되며, PDF 본문이 웹페이지로 직접 변환되어 있지 않다. 그러므로 고정된 참조 위치 정보가 필요하지 않으므로 검사 대상에서 제외된다.

### 2)	설치 애플리케이션
웹 페이지가 아닌, 별도의 뷰어 프로그램을 다운로드하여 설치 후 실행한 경우로, 웹 표준 기술을 사용하여 직접 제공된 전자출판문서가 아니므로 검사 대상이 아니다.
