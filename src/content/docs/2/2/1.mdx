<Aside title="검사 항목">
시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.
</Aside>

## 목적
시간제한이 적용된 콘텐츠를 제공할 경우, 사용자가 콘텐츠를 읽거나 양식을 제출하는 데 충분한 시간을 확보할 수 있도록 응답시간을 조절할 수 있는 기능을 제공해야 한다. 이를 통해 시각, 인지, 운동 등에 제약이 있는 사용자도 시간제한으로 인한 불이익 없이 웹 콘텐츠를 온전히 이용할 수 있도록 지원한다.

##	배경 및 필요성
예를 들어, 인증번호 입력과 같이 제한된 시간 내에 입력해야 하는 콘텐츠는 시각 장애, 인지 장애, 학습 장애 등을 가진 사용자에게 부담이 될 수 있다.

특히, 입력 시간이 오래 걸리거나 콘텐츠를 천천히 읽는 사용자는 제한된 시간 내 작업을 완료하기 어려워 오류가 발생하거나 정보를 놓칠 가능성이 높다.

화면낭독프로그램이나 키보드 사용자는 시간제한으로 인해 콘텐츠 변경이나 세션 종료 시 조작 시간이 더 걸리거나 주어진 시간 내 반응하기 어려울 수 있다. 이에, 사용자가 시간제한 기능을 해제하거나 연장할 수 있는 옵션을 제공하여, 사용자가 자신의 속도에 맞게 콘텐츠를 이용할 수 있도록 해야 한다.  

온라인 경매나 실시간 게임처럼 반응시간 조절이 원칙적으로 허용되지 않는 경우나, 세션 시간이 20시간 이상이면 예외로 본다. 이러한 경우에도, 사용자가 시간제한이 존재함을 미리 인지하고 종료 시점을 알 수 있도록 안내해야 한다.

##	기본 규칙
시간제한이 있는 콘텐츠를 제공해야 하는 경우, 다음 중 하나 이상의 기능을 제공해야 한다.
- 사용자가 시간제한 기능을 해제할 수 있는 설정을 제공해야 한다.
- 사용자가 남은 시간을 충분히 연장할 수 있는 기능을 제공해야 한다.
- 제한 시간을 연장할 수 있는 기능을 사용자가 쉽게 접근할 수 있도록 제공해야 한다. 제한 시간이 만료되기 최소 20초 전에 사용자에게 연장 가능 여부를 안내해야 한다. 사용자가 스페이스바 누르기 등 간단한 조작으로 시간제한을 10회 이상 연장하거나, 기본 제한 시간의 10배 이상을 추가로 확보할 수 있도록 해야 한다.
- 콘텐츠 시작 전에 시간제한이 적용됨을 미리 알리고, 제한 시간이 만료되면 종료됨을 안내해야 한다.
- 제한 시간이 조정 불가능한 경우(예: 온라인 경매, 실시간 게임, 20시간 이상 유지되는 세션)라도, 사용자에게 사전 안내를 통해 종료 시점을 명확히 알려야 한다.

## 용어
<dl>
	<dt>세션</dt>
	<dd>사용자가 웹사이트에 접속한 후 로그아웃하거나 연결이 끊길 때까지 유지되는 기간을 의미한다. 이 기간에 서버는 로그인 상태를 유지하고, 페이지 이동 간 정보를 공유할 수 있도록 사용자 데이터를 저장하고 관리한다.</dd>
	<dt>리다이렉트</dt>
	<dd>사용자가 특정 웹 페이지에 접근했을 때, 자동으로 다른 페이지로 이동시키는 기능을 의미한다.</dd>
</dl>

## 적용 사례
### 1)	시간 연장 기능 제공
본인 인증 과정에서 인증번호 입력 제한 시간이 설정되어 있으며, 제한 시간이 만료되기 전에 시간 연장 버튼을 클릭하여 시간을 여러 번 연장할 수 있도록 제공하였다.

### 2)	만료 전(세션 종료) 안내 및 연장 기능 제공
자동 로그아웃이 예정된 시점 1분 전에 팝업 창을 띄워 사용자에게 알림을 제공하고, 연장 버튼을 통해 세션 시간을 연장할 수 있도록 제공한 사례이다.

세션이 일정 시간이 지나 자동으로 만료되는 경우, 사용자가 작업 중인 내용을 잃지 않도록 사전에 안내해야 한다. 연장 버튼을 통해 세션 시간을 연장할 수 있도록 하여, 사용자가 세션 만료가 임박했음을 인지하고 필요한 경우 작업을 이어갈 수 있도록 한다.

## 개선 필요 사례

### 1)	인증 시간제한에 연장 기능 미제공
제한된 인증번호 입력 시간에 연장 기능을 제공하지 않은 사례이다.

시간제한이 설정되어 있지만, 사용자가 이를 연장하거나 해제하는 방법이 제공되지 않았다. 이로 인해, 화면을 읽는 데 시간이 더 필요한 시각장애 사용자나, 키보드만으로 입력해야 하는 운동 장애 사용자는 인증번호를 입력하지 못해 절차를 완료할 수 없게 된다.

결과적으로 사용자는 서비스 접근이 차단되거나, 인증 절차를 반복해야 하는 불편을 겪을 수 있다.

<figure>
	![](/images/image152.png)
	<figcaption>이미지. 인증번호 입력 시간제한에 연장 기능 미제공 사례</figcaption>
</figure>

#### 개선 방법
모든 사용자가 작업을 완료할 수 있도록 시간제한 해제 옵션을 제공하거나, 연장 기능을 추가해야 한다. 시간제한이 필수적인 경우, 사용자에게 이를 명확히 안내하고, 작업 실패 시 재시도 옵션을 제공하여 서비스 접근이 차단되지 않도록 해야 한다.

### 2)	접속 시간제한에 연장 기능 미제공
작성 중이던 댓글을 등록하려 할 때, 접속 시간이 만료되어 자동으로 로그아웃되면서 댓글이 등록되지 않은 사례이다.

시간제한을 연장할 수 있는 기능이 없어 사용자는 제한 시간이 존재하는 사실을 인지하지 못한 채 작업을 진행하다가 예기치 않게 로그아웃되는 상황을 겪게 된다.  

또한, 시간 연장 기능이 제공되지 않으면 제한 시간 내 작업을 완료하지 못한 사용자는 작업을 처음부터 다시 시작해야 한다.

	![](/images/image153.png)
	<figcaption>이미지. 접속 시간 만료로 댓글 등록이 실패하는 사례</figcaption>
</figure>

#### 개선 방법
접속 시간제한이 설정된 경우, 사용자에게 제한 시간이 있다는 사실을 명확히 안내해야 한다. 사용자가 페이지에 접속한 시점부터 남은 시간을 표시하거나, 일정 시간이 지나면 경고 메시지를 제공하여 제한 시간이 얼마나 남았는지 알릴 수 있어야 한다.

제한 시간이 만료되어도 작업 내용을 보존할 수 있도록, 자동 저장 기능을 추가하여 사용자가 작업을 처음부터 다시 시작하는 불편을 줄여야 한다.

### 3)	자동 전환 페이지에 전환 시간 연장 기능 미제공
특정 페이지(URL)에 접속했을 때 자동으로 다른 페이지로 이동시키는 리다이렉트 페이지가 너무 빠르게 전환되어, 사용자가 안내 내용을 충분히 읽을 수 없는 사례이다. 

페이지 이동에 대한 주의 및 안내가 제공되었지만, 사용자가 페이지 내용을 충분히 인지하고 읽을 시간이 제공되지 않았다. 리다이렉트가 너무 빠르게 이루어지면, 사용자는 안내 정보를 확인하지 못하고 페이지 내용을 파악하기도 어려워진다.

<figure>
	![](/images/image154.png)
	<figcaption>이미지. 자동 전환 페이지 인식에 충분한 시간이 제공되지 않은 사례</figcaption>
</figure>

#### 개선 방법
자동으로 페이지가 전환되는 시간을 충분히 늘려, 사용자가 안내 내용을 읽고 이해할 수 있는 시간을 제공해야 한다.

또한, 페이지 전환을 자동으로 수행하지 않고, 사용자가 직접 전환을 선택할 수 있는 버튼이나 링크를 제공하는 방법도 고려해야 한다. 이 경우, 사용자에게 안내 메시지를 명확히 전달하고 전환 버튼의 위치와 기능을 쉽게 알 수 있도록 해야 한다.
