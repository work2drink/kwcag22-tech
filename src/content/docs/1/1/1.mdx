<Aside title="검사 항목">
텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.
</Aside>

## 목적
대체 텍스트는 이미지, 차트, 다이어그램, 오디오 등 텍스트가 아닌 콘텐츠가 전달하는 의미나 용도를 인식할 수 있도록 제공하는 간결한 텍스트이다. 이를 통해 시각이나 청각 등의 감각에 제한이 있는 사용자도 동등한 정보를 얻을 수 있다.

##	배경 및 필요성
이미지를 볼 수 없는 사용자는 화면낭독프로그램이 읽어주는 대체 텍스트를 통해 내용을 이해할 수 있으며, 소리를 들을 수 없는 사용자는 화면에 표시된 대체 텍스트를 통해 내용을 확인할 수 있다.

대체 텍스트가 제공되면, 사용자 에이전트(브라우저 등)나 보조 기술이 다양한 방식으로 콘텐츠 정보를 사용자에게 제공할 수 있다. 이를 통해 시각, 청각 등 감각적 제약에 관계없이 모든 사용자가 동등한 정보를 얻을 수 있도록 돕는다.
##	기본 규칙
### 대체 텍스트 작성 시
- 콘텐츠의 의미와 기능을 정확하고 동등하게 전달할 수 있도록 작성한다.
- 불필요한 정보나 장황한 설명을 피하고, 콘텐츠 맥락을 고려하여 간결하고 명확한 표현을 사용한다.
- 이미지 주변 텍스트와 동일한 정보를 중복 제공하지 않는다.
- 이미지 대체 텍스트에 ‘~이미지’, ‘~사진’, ‘~아이콘’ 등의 불필요한 단어를 포함하지 않는다.

### 의미 있는 배경 이미지의 경우
- 배경 이미지가 중요한 의미를 전달하는 경우, CSS로 숨김 텍스트를 추가하거나 ARIA 속성 등을 활용하여 보조 기술이 콘텐츠의 의미를 인식할 수 있도록 대체 텍스트를 제공한다.

### 긴 설명이 필요한 경우
- 그래프, 차트 등과 같이 긴 설명이 필요한 콘텐츠는 해당 콘텐츠 근처에 자세한 설명 텍스트를 제공하거나, 페이지 내 설명을 aria-describedby 속성으로 연결하거나, figure 및 figcaption 요소를 활용하여 설명을 제공할 수 있다.
- 또는, 콘텐츠 상단에 별도의 설명 페이지 링크를 제공하여 사용자가 추가 정보를 확인할 수 있도록 하는 방법도 고려할 수 있다.
- longdesc 속성은 HTML5에서 제거되었으므로, DOCTYPE이 HTML5인 경우 사용할 수 없다.

### 기능을 가진 경우
- 버튼, 링크 등 상호작용 요소에 사용된 이미지는 해당 기능을 직관적으로 이해할 수 있는 단어로 제공한다. 예를 들어, 돋보기 아이콘은 ‘돋보기 아이콘’이 아닌 alt="검색"으로 작성한다.
- 동일 링크 내 이미지와 텍스트가 함께 있는 경우, 중복된 정보 제공을 피하고자 이미지는 alt 속성을 빈 값(alt=””)으로 제공해야 한다.

### 의미 없는 장식용 이미지인 경우

- 디자인 목적의 장식 이미지는 보조 기술에서 무시되도록 alt 속성을 빈 값으로 제공한다.    (예: `<img src="bg-pattern.jpg" alt="">`)

다음과 같은 경우에는 예외적으로 대체 텍스트를 제공하지 않거나 제한적으로 제공할 수 있다.

- 대체 콘텐츠로써 제공된 경우: 이미 다른 대체 수단(예: 수어 영상, 그래프 하단의 표 또는 텍스트 설명 등)으로 동일한 정보가 제공된 경우, 추가적인 대체 텍스트는 필요하지 않다. 예를 들어, 비디오 콘텐츠에 수어 영상이 포함된 경우 해당 수어 영상에 대한 별도의 대체 텍스트는 제공하지 않아도 된다.
- 생방송 콘텐츠: 생방송 비디오(또는 오디오)처럼 내용을 미리 알 수 없고, 지속적으로 변하는 경우, 해당 콘텐츠의 용도를 간략하게 설명하는 대체 텍스트만 제공한다.
- 시험이나 테스트: 시력검사, 청각검사 등과 같은 시험의 경우, 정답을 그대로 제공하면 안 되므로, 콘텐츠의 간략한 용도만을 설명하는 대체 텍스트를 제공한다.
- 감각적 경험을 제공하는 콘텐츠: 교향곡 연주, 예술 작품 등과 같이 말로 설명하기 어려운 감각적 경험을 제공하는 콘텐츠는, 콘텐츠의 이름과 가능한 경우 간략한 설명만 제공할 수 있다.
- CAPTCHA: 사람임을 증명하기 위한 테스트(예: CAPTCHA)에는 사람이 읽을 수 있으나 컴퓨터가 쉽게 해독할 수 없는 콘텐츠가 포함된다. 이 경우 대체 텍스트를 제공하면 CAPTCHA의 목적이 무력화될 수 있으므로, CAPTCHA의 목적과 사용 방법을 간단히 설명하는 텍스트만 제공하여 사용자가 테스트의 의도를 이해할 수 있도록 한다.
- 사용자 생성 콘텐츠: 사용자가 콘텐츠를 업로드할 때, 대체 텍스트를 입력할 수 있는 입력란을 제공하여 해당 콘텐츠의 내용을 제공할 수 있도록 유도한다.

## 적용 사례

### 1)	텍스트를 포함한 텍스트가 아닌 콘텐츠
이미지, 아이콘, 그래프, 오디오와 같은 텍스트가 아닌 콘텐츠에 텍스트를 포함하여 제공할 때, 해당 콘텐츠가 전달하는 의미나 용도를 모든 사용자가 동등하게 이해할 수 있도록 대체 텍스트를 제공해야 한다.

#### a)	짧은 텍스트가 포함된 이미지
다음은 이미지 내부에 짧은 텍스트가 포함된 사례이다. 이 경우, 시각적으로 전달되는 정보와 동등한 수준의 정보를 대체 텍스트로 제공해야 한다.

img 요소의 alt 속성을 활용하여 이미지에 포함된 텍스트와 동일한 내용을 제공하면, 화면을 볼 수 없는 사용자도 보조 기술을 통해 이미지가 전달하는 내용을 동등하게 전달받을 수 있다.

<figure>
  ![](/images/111/01.png)
  <figcaption>이미지. 짧은 텍스트를 포함한 이미지 배너 사례 (출처: 정부24)</figcaption>
</figure>

```html
<a href="...">
	<img src="" alt="정부24를 사칭하여 개인정보를 요구하는 스미싱문자 및 피싱사이트에 각별히 주의하세요.">
</a>
```

#### b)	긴 내용의 텍스트가 포함된 이미지

이미지 내부에 긴 텍스트가 포함된 경우, 주요 정보를 동등하게 전달해야 한다. 이때 사용자가 내용을 쉽게 이해할 수 있도록 표, 목록, 문장 형태로 구조화하여 제공할 수 있다. 이때, 콘텐츠의 맥락과 목적을 고려하여 정보 전달 방식을 결정해야 한다.

다음은 이미지 내부에 긴 텍스트가 포함된 경우의 사례이다. 화면낭독프로그램은 img 요소의 alt 속성을 한 번에 읽기 때문에, 사용자는 내용을 단계적으로 탐색하거나 필요한 부분만 선택적으로 읽는 것이 어렵다. 따라서 img 요소의 alt 속성만으로 긴 내용을 전달하는 것은 한계가 있다.

이러한 한계를 해결하기 위해, 숨김 텍스트를 활용하거나 별도의 상세 설명 페이지 링크를 제공하는 등 대체 방법을 고려할 수 있다.

<figure>
  ![](/images/111/02.png)
  <figcaption>이미지. 긴 텍스트를 포함한 이미지 배너 사례 (출처: 국민신문고)</figcaption>
</figure>

```css
.visually-hidden {
  clip-path: inset(100%);
  clip: rect(1px, 1px, 1px, 1px);
  height: 1px;
  overflow: hidden;
  position: absolute;
  white-space: nowrap;
  width: 1px;
}
```

```html
<img src="..." alt="국민신문고를 사칭하는 이메일에 주의하세요">
<p class="visually-hidden">
  신청한 적 없는 민원신청번호가 기재된 이메일은 열람하지 말고 삭제해주세요. <br>
  이메일을 열람한 경우 인터넷보호나라 등에 신고해주세요.(http://www.boho.or.kr)
</p>

```

### 2)	텍스트가 포함되지 않은 텍스트가 아닌 콘텐츠
텍스트가 포함되지 않은 텍스트가 아닌 콘텐츠의 경우, 콘텐츠가 전달하는 의미나 기능을 인식할 수 있도록 적절한 대체 텍스트나 설명을 제공해야 한다.

#### a)	SVG 아이콘으로 구현된 버튼
아이콘은 정보를 간결하고 직관적으로 전달하는 시각적 기호이다. 따라서, 사용자가 아이콘의 의미를 쉽게 이해할 수 있도록 간결하면서도 명확한 대체 텍스트를 제공해야 한다.

다음은 검색 기능을 나타내는 돋보기 아이콘을 SVG로 적용한 사례이다. 이 경우, 아이콘이 전달하는 ‘검색’이라는 의미를 대체 텍스트로 제공해야 한다. 숨김 텍스트, aria-label, SVG의 title 요소를 활용하여 대체 텍스트를 제공한 예시이다.

<figure>
  ![](/images/111/03.png)
  <figcaption>이미지. 검색을 표현한 돋보기 SVG 아이콘 사례 (출처: 다음)</figcaption>
</figure>

```html
<!-- 1.숨김 텍스트로 제공 -->
<button class="btn_search">
  <span class="visually-hidden">검색</span>
	<span class="ico_sch">
    <!-- IE에서 svg가 포커스 받는 것 방지가 필요한 경우focusable 속성 사용 -->
    <svg aria-hidden="true" focusable="false" width="28" height="28" viewBox="0 0 28 28" …>
		...
		</svg>
  </span>
</button>

<!-- 2.aria-label로 제공 -->
<button class="btn_search" aria-label="검색">
  <span class="icon_search">
		<svg focusable="false" width="28" height="28" viewBox="0 0 28 28"…>
		...
		</svg>
	</span>
</button>

<!-- 3.SVG 내 title 요소와 aria-labelledby로 제공 -->
<button class="btn_search">
	<span class="icon_search">
		<svg focusable="false" role="img" aria-labelledby="srch"  width="28" height="28" viewBox="0 0 28 28"…>
      <title id=”srch”>검색</title>
      ...
    </svg>
  </span>
</button>
```

이와 같이 SVG 아이콘에 대체 텍스트를 제공하는 방법은 여러 가지가 있다. 그러나 SVG 내 title 요소만으로 제공하는 방식은 iOS VoiceOver(iOS 18.1 기준)에서 이를 지원되지 않는다. 이를 보조 기술이 인식할 수 있도록 title 요소과 svg 요소를 aria-labelledby 속성으로 연결하여 보완할 수 있다.
이처럼 중요한 콘텐츠가 사용자에게 올바르게 전달되도록 하기 위해서는 보조 기술 지원 여부를 반드시 확인하고 테스트해야 한다.

#### b)	정보 전달을 위한 기호 이미지
다음은 진료 시간표에서 진료할 수 있는 일정을 나타내기 위해 동그라미 이미지를 제공한 사례이다. 이미지 자체로는 의미를 전달하지 않지만, 맥락을 통해 진료가 배정되어 있음을 나타내므로 대체 텍스트로 ‘진료 가능’을 제공한다.

3)	정보를 보완하는 이미지
이미지가 주변 텍스트와 콘텐츠의 맥락을 보완하는 역할을 하는 경우, 사용자는 이미지가 전달하는 의미를 충분히 파악할 수 있어야 한다. 따라서 이미지를 간결하면서도 정확히 묘사하는 대체 텍스트를 제공해야 한다.

다음은 래브라도 리트리버의 성격과 특성을 설명하는 텍스트를 보완하는 이미지 사례다.

이미지는 리트리버의 활발하고 놀이를 좋아하는 성격을 시각적으로 전달하며, 적절한 대체 텍스트는 화면낭독프로그램 사용자가 이미지의 의미와 콘텐츠 맥락을 이해하는 데 도움을 준다.

</figure>

```html
<img src="labrador.jpg" alt="잔디밭에서 달리며 오렌지색 장난감 하나를 함께 물고 있는 블랙과 골든 래브라도" ...>
```

### 4)	그래프, 차트 등 복잡한 의미를 전달하는 콘텐츠
데이터를 시각화한 그래프나 차트, 복잡한 관계를 표현하는 다이어그램 등은 대체 텍스트를 간단한 설명과 상세 설명으로 구분하여 제공할 수 있다. 간단한 설명은 시각적 요소의 유형과 목적을 요약하여 제공하고, 상세 설명은 별도의 텍스트를 활용하여 시각적 요소가 전달하는 모든 중요한 정보를 포함한다.

구체적으로, 데이터가 시각화된 그래프나 차트 등은 주요 추세, 최댓값과 최솟값, 특이점, 데이터 간의 관계를 설명해야 한다. 구조도 등의 다이어그램의 경우 구성 요소 간의 관계, 흐름, 계층 구조 등의 정보가 전달되어야 한다.

다음은 막대그래프 정보를 표와 함께 제공하여 그래프를 인식하기 어려운 사용자도 표를 통하여 이해할 수 있도록 한 사례이다.

### 5)	꾸밈을 위해 사용된 이미지

## ■	개선 필요 사례
### 1) `img`, `input type="image"` 등 이미지 요소에 alt 속성 미제공
다음은 배송 서비스를 홍보하기 위해 img 요소로 제공한 배너에 alt 속성이 제공되지 않은 사례이다. alt 속성이 누락된 경우, 화면낭독프로그램은 이미지 정보를 제대로 인식하지 못해 이미지의 URL 또는 파일명을 읽어 주거나, 단순히 ‘이미지’ 또는 ‘그래픽’이라고 낭독할 수 있다.

따라서 화면낭독프로그램 사용자는 이미지 정보를 인식할 수 없다.

<figure>
  ![](/images/111/15.png)
  <figcaption>이미지. 정보가 있는 이미지 alt 속성이 누락된 사례</figcaption>
</figure>

```html
<div class="">
	<a href="..."><img src="bn1.jpg"></a>
</div>
<div class="">
	<a href="..."><img src=" bn2.jpg"></a>
</div>
```

#### 개선 방법
이미지 요소에는 alt 속성을 사용하여 이미지가 전달하는 정보와 동등한 수준의 대체 텍스트를 제공해야 한다.
```html
<img src="bn1.jpg" alt="이커머스 배송의 필수 파트너 000 당일배송">
<img src="bn2.jpg" alt="원하는 시간, 원하는 만큼 자유로운 배송 알바 000 라이더스">
```

### 2)	의미 있는 배너 이미지에 대체 텍스트를 빈 값으로 제공
다음은 쇼핑 사이트에서 상품 주문 시 빠른 주문 방법을 안내하는 이미지를 제공하면서, 해당 이미지의 대체 텍스트를 빈 값으로 설정한 사례이다.

대체 텍스트가 빈 값(alt="")으로 제공된 이미지는 보조 기술에서 정보를 전달하지 않는 이미지로 인식되어, 사용자에게 이미지의 존재 여부조차 알리지 않는다. 이로 인해 화면을 볼 수 있는 사용자와 보조 기술 사용자 간에 인식하는 정보 차이가 발생한다.

<figure>
  ![](/images/111/16.png)
  <figcaption>이미지. 빠른 주문 안내 텍스트 이미지에 대체 텍스트가 빈 값으로 제공된 사례</figcaption>
</figure>

```html
<img src="..." alt="">
```

#### 개선 방법
`alt="상품 주문 코드 2948-234를 이야기 해주시면 빠르게 주문이 가능합니다!"` 와 같이 이미지가 전달하는 정보를 `alt` 속성에 제공한다.

```html
<img src="..." alt="상품 주문 코드 2948-234를 이야기 해주시면 빠르게 주문이 가능합니다!">
```

### 3)	이미지 정보와 다른 alt 속성의 대체 텍스트 내용
이미지가 전달하는 정보와 대체 텍스트의 내용이 서로 상이하게 제공된 사례이다.

이미지의 실제 내용은 ‘투명성을 위한 환경정보공개시스템’이지만 alt 속성이 ‘esg-green’으로 설정되어 보조 기술 사용자는 다른 정보를 제공받게 된다.

<figure>
  ![](/images/111/17.png)
  <figcaption>이미지. 배너 이미지의 대체 텍스트가 상이한 사례</figcaption>
</figure>

```html
<img src="..." alt="esg-green">
```

#### 개선 방법
`alt="투명성을 위한 환경정보공개시스템"`과 같이 이미지가 전달하는 정보와 동등한 대체 텍스트를 제공한다.

```html
<img src="..." alt="투명성을 위한 환경정보공개시스템">
```

### 4)	꾸밈 이미지에 대체 텍스트 제공
채용 공고 페이지에서 섹션 제목을 꾸미기 위해 사용된 아이콘 이미지의 사례이다.

이 아이콘은 단순한 꾸밈 요소임에도 불구하고 ‘icon’이라는 대체 텍스트가 제공되어, 보조 기술 사용자에게 불필요한 정보를 전달하고 있다.

예를 들어, 화면낭독프로그램은 이를 ‘icon, 이미지, 공통자격요건’과 같이 읽어 주므로, 사용자가 해당 이미지의 대체 텍스트가 잘못 작성되어 중요한 정보가 누락된 것인지, 제대로 제공된 것인지 판단하기 어렵게 된다.

<figure>
  ![](/images/111/18.png)
  <figcaption>이미지. 꾸밈 용도로 사용된 이미지에 대체 텍스트가 제공된 사례</figcaption>
</figure>

```html
<td><img src="..." alt="icon">공통자격요건</td>
```

#### 개선 방법
이미지가 정보를 전달하지 않고 단순한 꾸밈 용도로 사용된 경우, alt=""와 같이 빈 값을 제공하여 보조 기술이 이미지를 무시하도록 설정해야 한다.

```html
<td><img src="..." alt="">공통자격요건</td>
```

### 5)	관계 정보를 알 수 없는 대체 텍스트
다음은 기업의 조직도를 이미지로 제공한 사례이다. 조직도는 부서 간의 소속과 계층 관계 정보를 시각적으로 제공하지만, 다음 예시처럼 대체 텍스트가 이를 충분히 전달하지 않으면 보조 기술 사용자는 정보를 제대로 파악할 수 없다.

<figure>
  ![](/images/111/19.png)
  <figcaption>이미지. 계층 구조로 이루어진 조직도 이미지 사례</figcaption>
</figure>

```html
<img src="..." alt="안전보건 경영을 추진하는 조직도입니다. 대표이사 산하의 경영지원부문 총무팀과 산업안전보건위원회를 운영 및 관리하고 있습니다.">
```

#### 개선 방법
조직도의 소속이나 상하 관계를 명확히 표현하여 계층 구조를 이해할 수 있도록 설명하는 텍스트 정보를 제공해야 한다. 

설명이 길다면 간략한 대표 설명을 alt 속성에 제공하고, 상세 설명은 별도의 숨김 텍스트로 제공할 수 있다. 또는, figure와 figcaption 요소를 활용하거나 별도의 텍스트 영역을 통해 설명을 제공할 수 있다.

```html
<img src="..." alt="조직도">
<div class="visually-hidden">
	대표이사 아래에는 경영지원부문과 산업안전보건위원회가 있습니다. 경영지원부문에서는 산업 안전 대응 총괄, 주관 운영 업무를 담당하고, 산업안전보건위원회에서는 안전보건사항 논의, 심의의결, 협력 업무를 담당합니다. 경영지원부문 아래에는 총무팀이 있고, 안전/보건관리 업무 및 시설 관리를 담당합니다.
</div>
```

### 6)	img의 대체 텍스트를 title 속성만으로 제공
다음은 ‘지원하기’ 버튼을 이미지로 제공하면서, 대체 텍스트를 title 속성으로만 제공한 사례이다.

화면낭독프로그램 지원 또는 화면낭독프로그램의 설정에 따라 title 속성이 읽히지 않을 수도 있으며, 이로 인해 이미지 정보가 사용자에게 제대로 전달되지 않을 가능성이 높다.

<figure>
  ![](/images/111/20.png)
  <figcaption>이미지. 이미지 요소의 대체 텍스트가 title 속성으로만 제공된 예시</figcaption>
</figure>

```html
<img src="apply_btn.jpg" title="지원하기">
```

#### 개선 방법
이미지의 대체 텍스트는 alt 속성을 사용하여 제공해야 하며, title 속성은 추가적인 정보 제공 용도로만 활용해야 한다.
```html
<img src="apply_btn.jpg" alt="지원하기">
```
### 7)	정보가 포함된 이미지에 대체 텍스트 미제공
여러 장의 사진으로 구성된 뉴스 콘텐츠에서 모든 사진에 동일한 대체 텍스트를 제공한 사례이다. 이로 인해 화면낭독프로그램 사용자는 각 사진이 전달하는 개별적인 정보를 인식하지 못한다.

또한, title 속성에도 alt 속성과 동일한 내용이 제공되어 불필요한 중복이 발생하고 있다.

<figure>
  ![](/images/111/21.png)
  <figcaption>이미지. 여러 개의 사진 이미지에 모두 동일한 대체 텍스트 제공 사례</figcaption>
</figure>

```html
<img src="MDE=_5.jpg" title="국립단체 청년 교육단원 통합 발대식" alt="국립단체 청년 교육단원 통합 발대식">
<img src="MDI=_6.jpg" title="국립단체 청년 교육단원 통합 발대식" alt="국립단체 청년 교육단원 통합 발대식">
<img src="MDQ=_6.jpg" title="국립단체 청년 교육단원 통합 발대식" alt="국립단체 청년 교육단원 통합 발대식">
```

#### 개선 방법
각 사진의 내용을 설명하는 개별적이고 의미 있는 대체 텍스트를 제공해야 한다. 또한, alt 속성과 동일한 title 속성은 불필요하므로 제거하여 중복된 정보가 전달되지 않도록 한다.
```html
<img src="MDE=_5.jpg" alt="국립예술단체 청년 교육단원 통합 발대식 단체 사진">
<img src="MDI=_6.jpg" alt="문화체육관광부 장관의 개회사 발표 전경">
<img src="MDQ=_6.jpg" alt="장관, 기관 단장이 참여한 토론회 진행 모습">
```

### 8)	QR 코드 이미지에 용도 및 이용할 수 있는 정보 미제공
다음은 페이지에 QR 코드 이미지가 제공된 사례이다.

시각적으로 QR 코드를 인식할 수 있는 사용자는 주변 맥락을 통해 QR 코드의 용도를 추측할 수 있지만, 화면낭독프로그램 사용자는 그러한 정보를 얻기 어렵다.

<figure>
  ![](/images/111/22.png)
  <figcaption>이미지. QR 코드 이미지 사례</figcaption>
</figure>

```html
<img src="..." alt="QR Code">
```

#### 개선 방법
QR 코드의 용도 및 저장된 주소 정보를 대체 텍스트로 제공하거나, QR 코드를 링크와 함께 제공하여 사용자가 직접 해당 페이지로 이동할 수 있도록 한다.
```html
<a href="...">
	<img src="..." alt="QR 코드, 웹 접근성을 고려한 콘텐츠 제작 기법 바로가기">
</a>
```

### 9)	사용자가 등록하는 콘텐츠에 대체 텍스트 입력 기능 미제공
다음은 사용자가 선물 후기를 등록할 때 사진을 첨부하더라도, 해당 사진에 대한 대체 텍스트를 입력할 수 없는 사례이다.

사진 내용 없이 사진만 첨부될 경우, 화면낭독프로그램 사용자는 해당 사진의 내용을 알 수 없다.

</figure>

#### 개선 방법
사진이나 동영상을 첨부할 때, 사용자가 직접 설명을 입력할 수 있는 기능을 제공해야 한다. 참고로, 네이버 쇼핑 후기에서는 사진을 올릴 때 설명도 함께 작성할 수 있도록 제공하고 있다.

### 10)	이미지 맵에 대체 텍스트 미제공
다음은 이미지 맵을 사용하여 이미지의 특정 영역을 링크로 제공한 사례이다.

지도 이미지를 제공하고 각 시도별 영역을 나누어 링크를 제공하였으나 img 및 area 요소에 alt 속성이 빈 값으로 제공되었다. 이 경우, 이미지 맵 내 각 영역이 어떤 기능을 하는지 또는 어떤 콘텐츠로 연결되는지에 대한 정보가 제공되지 않으므로, 화면낭독프로그램 사용자는 해당 기능을 인식하고 사용할 수 없다.

<figure>
  ![](/images/111/25.png)
  <figcaption>이미지. 대한민국 지도에 이미지 맵으로 지역별 링크를 제공한 사례</figcaption>
</figure>

```html
<img src="..." alt="" usemap="#guide_01_Map">
<map name="guide_01_Map">
  <area shape="rect" alt="" coords="..." href="...">
  <area shape="rect" alt="" coords="..." href="...">
  ...
</map>
```

#### 개선 방법
먼저 img 요소의 alt 속성에 전체적인 목적과 컨텍스트를 설명하는 대체 텍스트를 제공해야 한다. alt=”대한민국 지도, 각 지역 선택 시 상세 정보로 이동” 제공하고, 각 area 요소에 alt=”경기도”와 같이 구체적인 정보를 제공한다.
```html
<img src="..." alt="대한민국 지도, 각 지역 선택 시 상세 정보로 이동" usemap="#guide_01_Map">
<map name="guide_01_Map">
  <area shape="rect" alt="서울특별시" coords="..." href="...">
  <area shape="rect" alt="경기도" coords="..." href="...">
  ...
</map>
```

### 11)	이모지로 잘못 된 대체 텍스트 제공
이모지를 아이콘처럼 사용하면서, 대체 텍스트 역시 이모지로 제공한 사례다. alt 속성값이 이모지로만 구성된 경우, 보조 기술 환경에 따라 이모지를 정상적으로 해석하지 못해 대체 텍스트가 없는 것으로 처리될 가능성이 있다.

따라서, 단순한 장식 목적이라면 alt=""(빈 값)를 사용하여 화면낭독프로그램이 불필요한 정보를 읽지 않도록 한다. 반면, 의미 전달이 필요한 경우에는 사용자가 쉽게 이해할 수 있는 문구로 직접 대체 텍스트를 제공해야 한다.

<figure>
  ![](/images/111/26.png)
  <figcaption>이미지. 이모지를 아이콘으로 사용한 사례</figcaption>
</figure>


```html
<div>
  <span class="c-jUooxn">
    <img alt=" " class="c-hakyQ" src="1f4aa.webp">
	</span>
	<p>건강검진 양도 신청<span>복리 후생</span></p>
</div>
```

#### 개선 방법
위 사례에서는 이모지가 단순한 장식 용도로 사용되고 있으므로, alt 속성값을 빈 값(alt="")으로 설정하여 보조 기술에서 읽히지 않도록 한다.

참고로, 이모지는 환경에 따라 다르게 읽힐 수 있다. 예를 들어,   (알통) 이모지는 Windows에서는  "알통", iOS는 "근육질의 구부린 팔뚝", Android는 "이두박근"과 같이 플랫폼별로 다른 방식으로 전달된다. 따라서, 이모지를 의미 전달의 핵심 요소로 사용할 경우 플랫폼별 차이를 고려하여 적절한 대체 텍스트를 제공해야 한다.

```html
<div>
  <span class="c-jUooxn">
    <img alt="" class="c-hakyQ" src="1f4aa.webp">
	</span>
	<p>건강검진 양도 신청<span>복리 후생</span></p>
</div>
```
