<Aside title="검사 항목">
표는 이해하기 쉽게 구성해야 한다.
</Aside>

## 목적
데이터 표에서 행과 열의 제목(제목 셀)과 내용(내용 셀)을 연결하여, 보조 기술 사용자도 [복잡한 표](../../glossary/complex-data-table) 형식 데이터를 쉽게 이해하고 탐색할 수 있도록 돕는 항목이다.

##	배경 및 필요성
화면낭독프로그램과 같은 보조 기술 사용자는 HTML의 선형 구조와 표의 구조를 기반으로 데이터를 순차적으로 탐색한다. 이때, 제목 셀과 내용 셀의 관계가 명확하게 정의되지 않으면, 데이터 간의 연관성을 파악하기 어렵고, 특정 셀이 어떤 제목 셀과 연결되는지 이해하기 어려워진다.

특히 보조 기술 사용자는 표를 인식하는 방식에서 일반 사용자와 차이가 있다. 화면낭독프로그램은 표를 읽을 때 다음과 같은 순서와 방법으로 데이터를 탐색한다.

- 표의 셀은 마크업 순서에 따라 순차적으로 읽히며, 사용자는 각 셀의 내용을 듣고 해당 셀이 어느 행과 열에 속하는지 위치 정보를 전달 받는다. 
- 특정 내용 셀을 읽을 때, 화면낭독프로그램이 해당 셀이 속한 열이나 행의 제목 정보를 함께 읽어 주어 사용자가 맥락을 파악할 수 있도록 한다.
- 화면낭독프로그램에는 표의 행과 열을 따라 셀 간 이동을 돕는 단축키가 있어 탐색을 지원하며, 한 페이지에 여러 개의 표가 있을 경우 표 단위로 탐색할 수 있다. 

만약 제목 셀과 내용 셀의 관계가 모호하거나 표 구조에 대한 추가 정보가 부족하다면, 화면낭독프로그램은 내용을 순차적으로 읽기 때문에 표가 가진 맥락과 의미를 제대로 전달하기 어려워진다.

##	기본 규칙
- 표는 table, caption, tr, th, td 등 표 전용 HTML 요소를 올바르게 사용한다.
- 제목 셀에는 th 요소를 사용한다. [복잡한 표](../../glossary/complex-data-table)에서도 내용 셀이 어떤 제목 셀과 연결되는지 알 수 있도록 scope 속성이나 headers, id 속성을 함께 설정한다.
- 표에는 caption 요소를 사용해 제목과 설명 정보를 제공한다. 이를 통해 화면낭독프로그램 사용자는 표 내용을 일일이 읽지 않아도 전체 맥락과 요점을 빠르게 파악할 수 있다.
- 레이아웃(화면 배치) 목적으로 테이블을 사용하는 것은 가급적 지양한다. 불가피한 경우, 의미 부여를 위한 th, caption 요소 및 summary, scope 속성은 사용하지 않는다.

## 적용 사례

### 1)	표 정보 제공
보조 기술이 표의 용도와 목적을 쉽게 파악할 수 있도록, 적절한 제목과 설명을 제공해야 한다.
#### a)	표의 제목 제공
표의 제목은 caption 요소를 사용하여 제공한다. caption에 작성하는 내용은 표의 용도와 목적을 전달해야 하며, 사용자가 표의 정보를 이해할 수 있도록 간결하고 직관적으로 작성해야 한다.

반면에, 표 상단에 h3 요소나 일반 텍스트로 제목을 배치하더라도, 보조 기술은 이를 표의 제목으로 인식하지 않는다. 따라서, 제목이 시각적으로 표시되었더라도, caption 요소를 제공해 보조 기술 사용자도 표의 내용을 빠르게 이해할 수 있도록 해야 한다.

```html
<!-- 좋은 예시 -->
<table>
<caption>2024년 1분기 부서별 매출 현황</caption>
  <thead>
    <tr>
<th scope="col">부서</th>
        <th scope="col">매출액</th>
    </tr>
  </thead>

<!-- 잘못된 예시 -->
<h2>2024년 1분기 부서별 매출 현황</h2>
<table>
<thead>
    <tr>
      <th scope="col">부서</th>
      <th scope="col">매출액</th>
    </tr>
</thead>
```
### b)	표의 설명 제공
표가 복잡하거나 특이한 구조를 갖는 경우, 간단한 요약 설명을 통해 어떤 행이나 열에서 어떤 내용을 찾을 수 있는지 안내하면 사용자가 표를 빠르게 이해할 수 있다.

HTML 5에서는 `summary` 속성이 지원되지 않으므로, `caption` 요소에 설명을 함께 작성하거나, `aria-describedby` 속성을 사용하여 설명 텍스트와 연결하거나, `figure`와 `figcaption` 요소를 활용하는 등의 대체 기법을 사용할 수 있다.

다음은 `caption` 요소에 요약 정보를 추가하여, 제목 셀이 2행으로 구성되고 일부가 병합된 [복잡한 표](../../glossary/complex-data-table)를 화면낭독프로그램 사용자가 이해할 수 있도록 설명을 제공한 사례이다.

![이미지. 제목 셀 구조가 복잡한 테이블 사례 (출처: 서울특별시-서울재정포털)](/images/image045.png)

```html
<table>
	<caption>
		<strong>세입현황표</strong><br>
		<p>
			세입과목, 전입누계, 기간중 수입액(수입액, 과오납 반환, 과목경정, 차액), 금일누계로 구성되어있습니다.
		</p>
	</caption>
...
```
## 2)	표 제목 셀과 내용 셀의 연결
보조 기술이 표에서 제목 셀과 내용 셀 간의 관계를 이해할 수 있도록 구현해야 한다. 이를 통해 화면낭독프로그램이 각 셀이 어떤 제목 셀과 연관되는지 올바르게 읽어 주어, 사용자가 표를 보다 쉽게 이해할 수 있도록 돕는다.
### a)	첫 행이 제목 셀로 구성된 표
첫 번째 행 전체가 제목 셀(th)로 구성된 기본적인 형태의 표라면, 화면낭독프로그램이 추가적인 접근성 속성(scope, id와 headers)을 생략하여도 구조를 전달할 수 있다.

![이미지. 첫 행이 제목으로 제공된 표와 코드 사례 (출처: 국민건강보험)](/images/image046.png)

```html
<table>
<caption>지역 임시 진료 안내</caption>
  <tr>
    <th>월</th>
    <th>일</th>
    <th>지역명</th>
    <th>대상</th>
    <th>의료진</th>
  </tr>
```

### b)	첫 열이 제목 셀로 구성된 표
첫 열이 제목 셀인 경우, th 요소에 scope=”row”를 사용한다. 이를 통해 보조 기술은 해당 열이 행 제목임을 인식하고, 내용 셀이 어떤 행에 속하는지 전달할 수 있다.

![이미지. 첫 열이 제목으로 제공된 표와 코드 사례 (출처: 한국장학재단)](/images/image047.png)

```html
<table>
<caption>대학생 재능봉사 캠프 사업의 연도별 참여 대학 현황</caption>
  <tr>
<th scope="row">2010학년도 <br> (22개교)</th>
    <td>강원대학교, 경북대학교, 경상대학교, ...</td>
  </tr>
  <tr>
    <th scope="row">2011학년도 <br> (32개교)</th>
    <td>강원대학교, 건국대학교, 경북대학교, 경상대학교, ...</td>
  </tr>
<tr>
    <th scope="row">2012학년도 <br> (45개교)</th>
    <td>강원대학교, 건국대학교, 경북대학교, 경상대학교, ...</td>
  </tr>
</table>
```

### c)	제목 셀이 두 줄 이상이거나 병합된 [복잡한 표](../../glossary/complex-data-table) 
하나의 행이나 열에 두 개 이상의 제목 셀이 포함된 표에서는, 각 제목 셀에 고유한 ID를 지정해야 한다. 제목 셀은 `<th id="고유ID값">` 형식을 사용하고, 내용 셀은 `<td headers="연결이 필요한 제목 셀의 id값">` 형식으로 연결한다.

아래 예시는 2개의 열이 각각 제목 역할을 하는 주가 정보 표로, 제목 셀의 ID를 내용 셀의 headers 속성으로 연결한 사례다.

![이미지. 제목 셀의 id를 내용 셀의 headers로 연결한 예시](/images/image048.png)

```html
<tr>
  <th id="prev-index">전일지수</th>
  <td headers="prev-index">727.66</td>
  <th id="volume">거래량</th>
  <td headers="volume">860,869천주</td>
</tr>
<tr>
  <th id="open-price">시가</th>
	<td headers="open-price">728.57</td>
	<th id="trade-amount">거래글</th>
	<td headers="trade-amount">7,132,228백만</td>
</tr>
```

## 개선 필요 사례

### 1)	표의 제목 미제공
다음은 인터넷 상품을 설명하는 표에서 표 제목이 제공되지 않은 사례다. 화면낭독프로그램 사용자는 표의 주제나 내용을 탐색하기 전에 이를 파악하기 어려워 어떤 정보가 제공되는지 알 수 없다. 특히 한 페이지 내 표가 여러 개 있을 경우 각각의 표를 구분하거나 맥락을 이해하는 데 어려움을 겪게 된다.

![이미지. 표의 제목이 제공되지 않은 사례](/images/image049.png)

```html
<div class="title_block">인터넷 구분</div>
<table>
  <tbody>
    <tr>
      <th>구분</th>
      <th>속도</th>
      <th>비고</th>
    </tr>
...
```

#### 개선 방법
표 상단에 제목 텍스트를 제공하더라도, 이는 HTML 구조에서 표의 제목으로 인식되지 않는다. 따라서 caption 요소를 사용하여 표의 제목을 명확히 제공해야 한다.

```html
<div class="title_block">인터넷 구분</div>
<table>
<caption>인터넷 상품 안내</caption>
  <tbody>
...
```

### 2)	표 제목을 display:none 으로 숨김
다음은 표 제목을 caption 요소로 제공하였지만, CSS에서 `display:none`으로 설정되어 화면뿐만 아니라 보조 기술에도 해당 내용이 전달되지 않는 사례이다.

![이미지. 표의 제목 정보를 포함한 caption 요소를 display:none으로 노출되지 않도록 설정한 예시](/images/image050.png)

```css
caption {
    display: none;
}
```

```html
<table border="1">
<caption>게시판 목록</caption>
<colgroup>...</colgroup>
  <thead>
      <tr>
          <th scope="col">번호</th>
          <th scope="col">제목</th>
          <th scope="col">카테고리</th>
          <th scope="col">작성자</th>
          <th scope="col">작성일</th>
          <th scope="col">조회</th>
          <th scope="col">평점</th>
      </tr>
  </thead>
  ...
```

#### 개선 방법
표의 제목이나 설명은 화면에 보이지 않더라도 보조 기술에서 접근할 수 있어야 한다.

`display:none`, `visibility:hidden`, `font-size:0` 등으로 숨기면 보조 기술에서도 해당 콘텐츠를 인식하지 못하므로 이러한 방식은 피해야 한다. 숨김 처리를 할 때는 보조 기술에서 올바르게 탐색할 수 있도록 적절한 CSS 기법을 사용하여 제공해야 한다.

앞서 다룬 적절한 대체 텍스트 제공 검사항목에서 제시한 `.visually-hidden` CSS 정의를 참조하여 보조 기술에서 인식할 수 있도록 구현한다.

### 3)	부적절한 표 제목
객실 정보 표에 ‘지리산 버스 투어 정보’라는 잘못된 표 제목이 적용된 사례다. 표의 제목과 실제 제공되는 정보가 다르기 때문에, 화면낭독프로그램 사용자는 표의 내용을 해석하는 데 혼동을 겪을 수 있다.

![이미지. 객실 정보 표에 ‘지리산 버스 투어 정보’로 제목이 잘못 적용된 예시](/images/image051.png)

```html
<caption>지리산 버스 투어 정보</caption>
```

#### 개선 방법
표의 제목은 표가 제공하는 내용을 명확히 이해할 수 있도록 ‘객실 정보’ 로 제공한다.

```html
<caption>객실 정보(유형별 기준 인원 및 1박 판매가)</caption>
```

### 4)	표 설명 미제공
다음은 여행 정보를 제공하는 웹 페이지에서 대상 도시의 정보를 표로 제공한 사례이다.

표 제목으로 ‘도시 정보’가 제공되었지만, 이는 표의 내용을 충분히 전달하지 못한다. 표 제목이 제공되었음에도 화면을 볼 수 없는 사용자는 이 표가 어떤 정보를 전달하는지, 각 열과 행이 무엇을 의미하는지 예측하기 어렵다.

![이미지. 제목만으로는 표의 내용을 충분히 예측하기 어려운 예시](/images/image052.png)

```html
<table>
	<caption>도시 정보</caption>
	...
```

#### 개선 방법
summary 속성에 해당 표에 대한 정보를 추가하여 ‘주요 여행 도시의 현지 시간, 환율, 콘센트 정보 안내’와 같이 간결하게 설명을 함께 작성할 수 있다. 

```html
<table summary="도시의 현재 시간, 환율, 콘센트 정보">
	<caption>도시 정보</caption>
	...
```

앞서 설명한 것처럼, HTML5에서는 summary 속성이 제거되었으므로, caption 요소에 설명을 포함하거나 aria-describedby 속성을 이용하여 하는 등 다양한 대체 방법을 통해 보조 기술에 정보를 전달할 수 있다.

```html
<p id="tbl-cityinfo">각 도시의 현재 시간, 환율, 콘센트 정보</p>
<table aria-describedby="tbl-cityinfo">
	<caption>도시 정보</caption>
	...
```

### 5)	제목 셀 미제공
콘서트 정보 표에서 시각적으로는 제목 행이 보이지만, 마크업에서 해당 행에 th 요소를 사용하지 않아 보조 기술이 이를 제목 셀로 인식하지 못하는 사례다. 이로 인해 화면낭독프로그램 사용자는 제목 셀과 내용 셀 간의 정보 연관성을 파악하기 어려워지고, 특정 정보를 빠르게 찾거나 표 전체를 이해하는데 어려움을 겪을 수 있다.

![이미지. 콘서트 일정 표 예시](/images/image053.png)

```html
<table>
<caption>2024년 12월 콘서트 정보</caption>
  <tr>
    <td>날짜</td>
    <td>콘서트 이름</td>
    <td>가수</td>
  </tr>
  <tr>
    <td>2024-12-05</td>
    <td>2024 월드투어 - 서울</td>
    <td>아이브 (IVE)</td>
</tr>
...
```

#### 개선 방법
첫 행이 제목 셀인 경우, 제목 셀(th 요소)에 scope 속성을 사용하여 `<th scope="col">` 형식으로 제공하고 제목 행 전체는 thead 요소를 함께 사용하면 구조를 더 명확히 할 수 있다.  

제목인 첫 행에 th 요소를 부여하고, 제목 행 전체를 thead 요소로 감싸면 구조를 더욱 명확히 표현할 수 있다.

```html
<table>
  <caption>2024년 12월 콘서트 정보</caption>
  <thead>
    <tr>
      <th>날짜</th>
      <th>콘서트 이름</th>
      <th>가수</th>
    </tr>
  </thead>
	<tbody>
	<tr>
      <td>2024-12-05</td>
      <td>2024 월드투어 - 서울</td>
      <td>아이브 (IVE)</td>
    </tr>
		...
```

### 6)	표 안에 표 제공 (중첩된 테이블)
지역별 축제 정보를 담은 표에서, 내용 셀에 기념품샵 정보를 추가로 제공하기 위해 중첩된 테이블을 사용한 사례다.

중첩 테이블은 시각적으로는 구분하기 쉽지만, 순차적으로 탐색하는 보조 기술 입장에서는 표의 계층 구조가 복잡해 질 수 있다. 예를 들어, 화면낭독프로그램은 표의 행, 열, 셀에 대해 정보를 제공하지만, 중첩된 표가 있으면 제목 셀과 내용 셀 사이의 관계를 정확히 해석하지 못할 수 있다. 

또한, 셀 간 이동 방식이 예상과 달라져 탐색 명령이 의도대로 작동하지 않을 가능성이 높아진다.

![이미지. 지역별 축제 정보 표 안에 기념품샵 정보 표가 포함된 예시](/images/image054.png)

#### 개선 방법
가능하면 중첩 구조를 없애고 하나의 표로 표현한다. 하나의 표로 제공하기 어려운 경우, 중첩된 표를 분리하여 제공하거나 목록 요소 등을 활용할 수 있다.

### 7)	레이아웃 용도로 사용된 표에 제목, 설명, 제목 셀 사용
다음은 도서전 이벤트 페이지에서 콘텐츠 배치를 위한 레이아웃 용도로 table 요소를 사용한 사례이다.

데이터 표가 아님에도 표 제목이나 설명을 제공하면, 화면낭독프로그램이 이를 데이터 표로 인식하여 불필요한 설명을 제공하고 사용자는 해당 테이블을 데이터 표로 오해할 가능성이 있다. 이로 인해 사용자는 콘텐츠를 정확히 이해하지 못하거나 불편을 겪게 된다.

![이미지. 콘텐츠를 배치하는데 사용된 표 사례](/images/image055.png)

```html
<table>
<caption>도시전 2024 안내</caption>
<tbody>
    <tr>
      <th>도시전 2024</th>
    </tr>
    <tr>
      <td><img src="..." alt="..."></td>
    </tr>
    <tr>
      <td><p>"도시전 2024"는 12월 30일부터 12월 31일까지... </p><td>
    </tr>
    ...
  </tbody>
</table>
```

#### 개선 방법
콘텐츠 배치를 위한 레이아웃을 구현할 때는 table 요소 대신 적절한 HTML 요소와 CSS를 사용해 구조화하는 것이 바람직하다. 불가피하게 table 요소를 사용해야 하는 경우(예: 메일 발송 전용 콘텐츠 등), 데이터 표로 인식되지 않도록 th, caption 요소 및 summary, scope 속성과 같이 의미를 가지는 정보는 제공하지 않으며, 가능한 간단하게 작성해야 한다.

```html
<table>
	<tr>
    <td>도시전 2024</td>
  </tr>
  <tr>
    <td><img src="..." alt="..."></td>
  </tr>
  <tr>
    <td><p>"도시전 2024"는 12월 30일부터 12월 31일까지... </p><td>
  </tr>
	...
</table>
```
